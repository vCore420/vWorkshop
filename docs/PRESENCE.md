# Presence

> "A place does not become meaningful because it reacts to you.
>
> It becomes meaningful because it continues existing even when you are not there."

Throughout Version 2, the Workshop learned how to remember.

Projects persist.

Furniture remains where it was placed.

Time passes.

Weather changes.

Music continues.

Bubble wanders.

The world quietly continues existing.

These systems create persistence.

But persistence alone is not the destination.

The destination is presence.

## What is Presence?

Presence is the feeling that the Workshop exists independently of its visitor.

The Workshop should never feel like it has been constructed around the user.

Instead, the user should feel like they have entered somewhere that was already alive.

Nothing should exist solely because the user opened a menu.

Nothing should happen only because the player looked at it.

The Workshop should increasingly feel comfortable existing by itself.

## Returning

One of the most important moments in the Workshop is not opening a project.

It is arriving.

Returning after hours.

Returning tomorrow.

Returning next week.

The Workshop should greet people in the same way real places do.

Not with announcements.

Not with rewards.

Not with pop-ups.

Simply by still being there.

The sunlight is different.

The weather has changed.

Bubble is somewhere else.

The radio is still quietly playing.

A lamp was left on.

A project is still sitting on the Workbench waiting to continue.

Nothing announces your return.

Yet everything acknowledges it.

## Presence over Interaction

A common temptation in software is to make every system react to the user.

The Workshop should resist this.

Not every object needs interaction.

Not every moment needs feedback.

Not every action requires acknowledgement.

Sometimes the strongest design choice is allowing the Workshop to continue quietly being itself.

## Presence over Simulation

The Workshop does not need to simulate every aspect of reality.

It only needs enough consistency that the mind naturally fills in the rest.

Small habits.

Subtle continuity.

Meaningful persistence.

Quiet atmosphere.

These often create a stronger sense of place than large simulated systems.

## Future Guidance

When designing new features ask:

Does this make the Workshop feel more present?

Would this still make sense if nobody was watching?

Does this strengthen the feeling that the Workshop exists independently of the player?

If the answer is yes, it probably belongs.

If the answer is no, consider whether the feature is serving the Workshop, or merely adding complexity.

## Closing Thought

The goal of the Workshop has never been to imitate software.

It has always been to create somewhere people enjoy returning to.

A place does not become memorable because it constantly asks for attention.

It becomes memorable because, each time you return, it quietly reminds you that it never stopped being itself.

A reminder.

A question for future contributors.
